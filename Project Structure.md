# Sorter-Помогатор — описание проекта

> Этот файл создан для быстрой ориентации в проекте. Он описывает структуру файлов, логику работы и все режимы приложения.

---

## Что это за проект

Веб-приложение для управления задачами по методологии GTD (Getting Things Done). Состоит из двух частей:

1. **sorter 2025** — основное рабочее приложение (HTML + Vanilla JS, без фреймворков).
2. **Twine Pomogator** — прототип алгоритма GTD-разбора, написанный в Twine (интерактивная художественная программа). Является прадедом логики `process.html`.

Деплой: `https://item-sorter.pages.dev/` (Cloudflare Pages).

---

## Страницы и их роль

### `index.html` + `app.js` — Главный редактор

Основной экран. Большой `<textarea>` для списка задач в формате todo.txt. Все задачи хранятся в `localStorage`.

**Группа «Задача»:**
| Кнопка | Функция |
|---|---|
| ✅ Отметить выполненной | `toggleDoneCurrentLine()` — ставит/снимает `x DATE` на текущей строке курсора |
| 🏆 Присвоить приоритеты | `assignPrioritiesAfterSort()` — расставляет `(A)(B)(C)…` сверху вниз |
| ⚙ Обработать (GTD) | Открывает `process.html?task=…` с текущей строкой |
| ⚡ Быстрый разбор | `quickTriageCurrentTask()` — упрощённый КОРЗИНА-флоу прямо на странице через SweetAlert2 |

**Группа «Сортировка»:**
| Кнопка | Функция |
|---|---|
| Filter Tasks | `filterTasksUI()` — для каждой задачи показывает диалог: Включить / Игнорировать / Удалить |
| Sort Tasks | `sortTasks()` — сортирует задачи по важности через серию парных сравнений |
| Insert Unsorted Tasks | Вставляет новые задачи в уже отсортированный список (бинарная вставка) |
| Merge Arrays | `mergeArraysUI()` — сливает два блока (разделённых маркером `NEW ARRAY`) |

**Группа «Данные»:**
| Кнопка | Функция |
|---|---|
| Copy | Копирует задачи в буфер (без пустых строк) |
| Save .txt | Скачивает файл `tasks_ДАТА_ПЕРВАЯ-ЗАДАЧА.txt` |
| Войти в Dropbox | OAuth PKCE авторизация |
| В Dropbox / Из Dropbox | Сохранить/загрузить `/tasks.txt` в Dropbox |

### `process.html` — GTD-Помогатор

Открывается с параметром `?task=...` (URL-encoded текст задачи). Реализует два флоу:

#### Флоу «📥 Разбор корзины» (`korz`)

```
С этим надо что-то делать?
├── НЕТ → Мусор / Когда-нибудь / Справочная информация
└── ДА
    └── Мне? 
        ├── НЕТ → Делегировать (кому? → @имя в теге)
        └── ДА
            └── Сейчас / на этой неделе?
                ├── Конкретная дата → В календарь (due:ДАТА)
                ├── Когда-нибудь потом → someday
                └── ДА
                    └── Задача мутная?
                        ├── ДА → переключиться на флоу «mutny»
                        └── НЕТ
                            └── < 2 минут?
                                ├── ДА → Сделать сейчас
                                └── НЕТ → Следующее действие (next_action)
```

#### Флоу «🔮 Мутная задача» (`mutny`)

```
1. Какой результат должен быть? [textarea]
2. Запиши ОДНО следующее действие [textarea]
3. Есть глагол-прокрастиноген? (подумать / изучить / сделать…)
   ├── ДА → Переписать → вернуться к п.2
   └── НЕТ
       └── Приоритет? (🔴 Срочно / 🟢 Обычный)
           └── Контексты? (@дом / @работа / @компьютер…)
               └── → Вывод готовой строки todo.txt
```

**Результат** (`action` типы):
- `trash` — удалить
- `someday` — добавить тег `someday:yes`
- `ref` — добавить тег `ref:yes`
- `delegate` — добавить тег `@имя`
- `calendar` — добавить `due:ДАТА`
- `now` — пометить выполненным `x DATE`
- `next_action` — сформировать строку todo.txt и предложить скопировать или добавить в список

### `tasks.html` — Таск-трекер

Визуальный список всех задач из того же `localStorage`. Читает задачи в формате todo.txt.

**Возможности:**
- Быстрое добавление задачи (строка ввода вверху)
- Поиск по тексту
- Фильтрация по вкладкам: Все / Активные / Выполненные
- Фильтрация чипами по приоритету `(A–D)`, проектам `+proj`, контекстам `@ctx`
- Сортировка по приоритету / дате создания / дате due
- Отметить задачу выполненной чекбоксом
- Кнопка GTD-разбора на каждой строке (открывает `process.html`)
- Секция «игнорируемых задач» (свёрнута, отображается пунктиром)

## Формат данных — todo.txt

Задачи хранятся в `localStorage` под ключом `tasks` в формате [todo.txt](https://github.com/todotxt/todo.txt):

```
(A) 2026-04-01 Позвонить клиенту @телефон +проект due:2026-04-20
x 2026-04-15 2026-04-10 Сдать отчёт +работа
```

**Специальные маркеры-разделители в тексте:**
| Маркер | Смысл |
|---|---|
| `ИГНОРИРУЕМЫЕ ЗАДАЧИ ГГГГ.ММ.ДД` | Всё ниже — проигнорированные при фильтрации задачи |
| `NEW ARRAY` | Граница между двумя блоками при слиянии (`Merge Arrays`) |
| `НЕУПОРЯДОЧЕННЫЕ ЗАДАЧИ` | Хвост при трёхчастном разборе массива |

## Алгоритмы сортировки

Сортировка основана на **попарных сравнениях задач пользователем**. Функция `compareTasks(task1, task2)` показывает диалог SweetAlert2 с двумя кнопками и возвращает Promise с `-1` (task1 важнее) или `1` (task2 важнее).

| Алгоритм | Где используется | Особенности |
|---|---|---|
| `mergeSort` | Sort Tasks | Стандартное асинхронное слияние, O(n log n) сравнений |
| `mergeGalloping` | Merge Arrays | Слияние с галоп-оптимизацией (ускорение при длинных сериях) |
| Бинарная вставка | Insert Unsorted Tasks | O(log n) сравнений для вставки каждой новой задачи |

## Подсветка и фильтр-бар (`todotxt.js`)

- Поверх `<textarea>` рендерится прозрачный HTML-слой с подсветкой (`hl-layer`)
- Подсвечиваются: приоритеты `(A–Z)`, теги `key:value`, проекты `+proj`, контексты `@ctx`, даты, просроченные `due:`
- Фильтр-бар над полем автоматически собирает все проекты, контексты, приоритеты из текста и рендерит чипы-фильтры
